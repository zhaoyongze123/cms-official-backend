# 安全策略

## 支持范围

当前项目仍处于快速迭代阶段，默认只维护 `main` 分支的最新版本。

## 报告安全问题

请不要在公开 Issue 中披露漏洞细节、密钥、Token 或服务器访问方式。建议通过私有渠道联系仓库维护者，并提供：

- 受影响的模块或接口
- 可复现步骤
- 影响范围
- 建议修复方向

## 凭证处理

- `.env`、`.env.prod`、SSH 私钥、API Key、数据库密码不得提交到 Git。
- GitHub Actions 使用 `PROD_ENV_FILE` Secret 下发生产环境变量。
- 已泄露的凭证应立即轮换，不应继续用于开发或生产环境。
